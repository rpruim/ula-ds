Solution 5.1.4.9.1.

  1. If \(A \xvec = \lambda \xvec\) and \(\xvec \in \real^n\text{,}\) then \(A\) is associated with a linear transformation \(T: \real^n \to \real^n\) since both \(\xvec\) and \(\lambda \xvec\) are \(n\)-dimensional.
  2. If \(A\) is \(m \by n\text{,}\) then \(A\) is associated with a linear transformation \(T: \real^n \to \real^m\text{,}\) so we can’t use the same vector on both sides of \(A \xvec = \lambda \xvec\) since the dimensions don’t work out. But would could look for a \(\lambda\) and a pair of vectors \(\vvec\) and \(\uvec\) such that \(A \vvec = \lambda \uvec\text{.}\) In isolation is this is meaningless, since we can take \(\lambda = 1\) and \(\uvec = A \vvec\text{.}\) It becomes more interesting, however, if we look for two orthogonal bases (bases where the angles between any pair of basis vectors is 90 degrees), an orthogonal basis \(\{ \vvec_1, \vvec_2, \dots, \vvec_r \}\) of \(\row(A) = \col(A^\transpose)\) and an orthogonal basis \(\{ \uvec_1, \uvec_2, \dots, \uvec_r \}\) of \(\col(A)\text{.}\)
    It turns out that every matrix has such a set of bases. This will be our topic of discussion in Section 7.5 and Section 7.6.
